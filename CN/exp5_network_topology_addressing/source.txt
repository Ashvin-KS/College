2. Classful IP Configuration
The addressing for this stage follows standard Class A rules using the first octet derived from
my name.
Procedure for Classful IP Configuration
Step 1: Determine First Octet
Identify the ASCII value of the first letter of the name ('A' = 65) to use as the first octet for all IP
addresses.
Copper Straight-Through cables from PC and Laptop to Wireless Router's LAN ports
Addressing Logic: Since my name starts with 'A', the first octet is set to 65.
Subnet Mask: A default mask of 255.0.0.0 was used.
Device Settings:
PC: IP 65.45.34.25, Gateway 65.45.34.1.
Laptop: IP 65.45.34.26, Gateway 65.45.34.1.

Step 2: Configure Wireless Router
Set the router's IP address to 65.45.34.1 with subnet mask 255.0.0.0 to serve as the
network gateway.
Step 3: Configure PC
Assign IP address 65.45.34.25 with subnet mask 255.0.0.0 and default gateway
65.45.34.1.
Step 4: Configure Laptop
Assign IP address 65.45.34.26 with subnet mask 255.0.0.0 and default gateway
65.45.34.1.
Step 5: Verify Configurations
Use ipconfig command on both PC and Laptop to confirm IP addresses, subnet masks, and
gateway settings are correct.
Step 6: Test Connectivity
Perform ping tests between PC and Laptop, and from each device to the gateway to verify
network connectivity.
Step 7: Document Configuration
Record all IP addressing details and save the configuration as 24BCE1580_Classful.pka.
3. Classless IP Configuration
To demonstrate classless addressing (CIDR/Subnetting), the subnet mask was modified while
maintaining the same IP range.
Addressing Logic: The first octet remains 65.
Subnet Mask: The mask was manually changed to 255.255.255.0 to simulate a /24 subnet.
Device Settings:
PC: IP 65.45.34.25, Mask 255.255.255.0, Gateway 65.45.34.1.
Laptop: IP 65.45.34.26, Mask 255.255.255.0, Gateway 65.45.34.1.

Procedure for Classless IP Configuration
Step 1: Understand CIDR
Recognize that classless addressing allows flexible subnet mask allocation, enabling more
efficient IP address utilization compared to classful addressing.
Step 2: Modify Subnet Mask
Change the subnet mask from 255.0.0.0 to 255.255.255.0 (/24) on all devices while
keeping the same IP addresses.
Step 3: Reconfigure Router
Update the router's subnet mask to 255.255.255.0 while maintaining IP address 65.45.34.1.
Step 4: Reconfigure PC
Update the PC's subnet mask to 255.255.255.0 while maintaining IP address 65.45.34.25
and gateway 65.45.34.1.
Step 5: Reconfigure Laptop
Update the Laptop's subnet mask to 255.255.255.0 while maintaining IP address
65.45.34.26 and gateway 65.45.34.1.
Step 6: Verify New Configurations
Use ipconfig to confirm the updated subnet masks on all devices.

Step 7: Test Connectivity
Perform ping tests to ensure connectivity is maintained with the new subnet mask configuration.
Step 8: Compare Configurations
Observe how the /24 subnet creates a smaller, more manageable network compared to the /8
classful configuration.
Step 9: Document Configuration
Record the classless addressing details and save as 24BCE1580_Classless.pka.
